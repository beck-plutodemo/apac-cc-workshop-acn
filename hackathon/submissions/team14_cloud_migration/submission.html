The Problem

Before — The 2am Cron

  • Batch job fires at 02:00 every night regardless of data arrival
  • Processes entire reconciliation file even if nothing changed
  • Failures are silent until morning stand-up
  • Tight coupling: downstream consumers poll for output at fixed times
  • Scaling means running bigger VMs, not smarter pipelines

After — Event-Driven

  • Job triggers on file arrival in object storage
  • Processes only what arrived — nothing wasted
  • Failures surface immediately via event queue
  • Downstream consumers use compatibility shim — zero breakage
  • Horizontal scale via container replicas

Architecture

File lands in object storage → event fires → container processes → output lands downstream.

Event Trigger Logic

Worker exposes a lightweight webhook endpoint. MinIO (and S3/GCS) POST to it on every object creation event.

from fastapi import FastAPI, Request
from src.reconciler import run_reconciliation
from src.storage import get_storage_client

app = FastAPI()

@app.post("/trigger")
async def handle_event(req: Request):
    body = await req.json()

    # Normalise event shape across MinIO / S3 / GCS
    file_key = parse_file_key(body)

    if not file_key.endswith(".csv"):
        return {"status": "skipped", "reason": "not a recon file"}

    storage = get_storage_client()          # respects STORAGE_BACKEND env var
    data = storage.download(file_key)

    result = run_reconciliation(data)

    storage.upload(output_key(file_key), result)
    notify_downstream(file_key)             # compatibility shim call

    return {"status": "ok", "processed": file_key}

Compatibility Shim

Downstream consumers expect output at recon-processed/YYYYMMDD_recon_out.csv by 06:00. We keep that contract.

# src/compat_shim.py
"""
Ensures downstream consumers that poll for output files see
the same path/filename convention as before the migration.
"""
import os
from datetime import datetime

def notify_downstream(source_key: str):
    date_str = datetime.utcnow().strftime("%Y%m%d")
    legacy_path = f"recon-processed/{date_str}_recon_out.csv"

    # Write a pointer / symlink object so legacy pollers find their file
    storage = get_storage_client()
    storage.copy(output_key(source_key), legacy_path)

    # Optional: POST to legacy webhook if consumers subscribed that way
    legacy_webhook = os.getenv("LEGACY_WEBHOOK_URL")
    if legacy_webhook:
        requests.post(legacy_webhook, json={"file": legacy_path, "status": "ready"})

Zero changes required by downstream teams. They keep polling the same path. We stop caring about what time it is.

Storage Abstraction Layer

Single interface, three backends. Swap via STORAGE_BACKEND env var.

# src/storage.py
def get_storage_client():
    backend = os.getenv("STORAGE_BACKEND", "minio")
    match backend:
        case "minio":
            return MinIOClient(
                endpoint=os.getenv("STORAGE_ENDPOINT"),
                access_key=os.getenv("MINIO_ROOT_USER"),
                secret_key=os.getenv("MINIO_ROOT_PASSWORD"),
            )
        case "s3":
            return S3Client(bucket_prefix=os.getenv("S3_BUCKET_PREFIX"))
        case "gcs":
            return GCSClient(project=os.getenv("GCP_PROJECT"))
        case _:
            raise ValueError(f"Unknown storage backend: {backend}")

Hardcoded Paths

VM-Specific Config

How to Run It

Assumes Docker is installed. Nothing else required.

# 1. Start MinIO + worker
docker compose up -d

# 2. Create input bucket and configure webhook trigger
docker compose exec minio mc alias set local http://localhost:9000 minioadmin minioadmin
docker compose exec minio mc mb local/recon-incoming
docker compose exec minio mc mb local/recon-processed
docker compose exec minio mc event add local/recon-incoming \
  arn:minio:sqs::PRIMARY:webhook --event put

# 3. Drop a file in — watch it process
cp tests/fixtures/sample_recon.csv /tmp/
docker compose exec minio mc cp /tmp/sample_recon.csv local/recon-incoming/

# 4. Check output
docker compose exec minio mc ls local/recon-processed/

Switch to S3: set STORAGE_BACKEND=s3 and the relevant AWS env vars. Same image.

Key Decisions

Webhook over message queue

Simpler for local dev. MinIO supports both. Production can swap to SQS/Pub-Sub without changing worker code.

Compatibility shim over consumer migration

We don't own downstream. Shim lets us ship without a cross-team sprint. Remove it in Q3 once consumers migrate.

Single container, no orchestrator

Docker Compose is enough for the hackathon. K8s deployment YAML is scaffolded but not tested.

Idempotent processing

Worker checks output bucket before processing. Re-delivered events are safe — no double-reconciliation.
